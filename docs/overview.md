# Overview

**Jal** turns customer feedback into merged GitHub pull requests — with a UI nobody else ships.

## The problem

Product teams lose feedback in spreadsheets, Slack, and ticket graveyards. Engineers rebuild context from scratch. Customers never know if their request shipped.

## The solution

```text
Customer widget  →  Feedback River  →  AI pipeline  →  GitHub PR  →  Merge
```

## Three ways to use Jal

| Mode | Best for | Start here |
|------|----------|------------|
| **Jal Studio** | Fastest — browser UI, widget, river | [Get started](/docs/get-started) |
| **npm package** | Your backend, your routes | [npm](/docs/npm) |
| **Vendo app** | See Jal on a real product | `/internal/login` |

## Key concepts

**Feedback River** — Requests are droplets on an animated stream, not rows in a grid. Position = pipeline stage.

**Dual Lens** — Customer view (widget) and ship view (PR, tasks) side by side.

**Repo Pour** — Attach GitHub once; AI absorbs README, stack, and product context.

**Phosphor Abyss** — Color follows lifecycle: cold intake → violet depth → warm ship.

## Demo in 60 seconds

1. Homepage → scroll to `#river` → click a droplet
2. `/studio/onboard` → attach repo
3. `/studio/projects/:id` → ship from the river

→ Continue with [Get started](/docs/get-started)
